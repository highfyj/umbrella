# 剧情脚本 DSL 设计文档 v0.1

> 状态：草案。配套样例剧本见 `story/`，可作为本文档的"可执行用例"。

## 1. 目标与原则

- **写手优先**：90% 的内容是台词与旁白，这部分的书写体验必须接近写剧本，而不是填数据结构。
- **全程语音是一等公民**：每句配音台词都是被管理的资产，脚本系统同时承担录音台本管理职责。
- **资产由 AI 生成驱动**：立绘按"维度寻址的整图变体"组织，编译器枚举剧本实际用到的组合并输出缺失清单，作为 AI 出图工作列表（与录音台本对称）。
- **脚本先行，资产解耦**：缺失资产（图、音、语音）**永远只警告、不报错**，运行时用占位实现兜底——剧本写作与调试在零资产状态下即可全流程进行。报错只留给"引用了未注册的名字"这类拼写错误。
- **编译期尽早报错**：结构错误（Schema）、语义错误（linter）、资产/语音缺失全部在编译期暴露，并精确到源文件行列。
- **运行时确定性**：运行器是一个确定性虚拟机（含随机数），存档 = 序列化 VM 状态。

## 2. 总体管线

```
story/*.yaml ──解析(保留行列位置)──► 结构校验(JSON Schema)
            ──语义检查(linter)──► 编译 ──► IR(JSON) ──► 运行时 VM
                                    │
                                    ├─► 录音台本导出（给配音）
                                    ├─► 立绘缺失组合清单（给 AI 出图）
                                    ├─► voice.lock 三向核对
                                    └─► 剧情流程图（只读可视化）
```

运行时只认 IR，发布包不含 YAML 解析器与校验器。

## 3. 文件组织

```
story/
  story.yaml          # 入口：元信息、变量声明、结局注册
  characters.yaml     # 角色注册表（含立绘变体注册）
  assets.yaml         # 资源清单（背景/BGM/SE；立绘归 characters.yaml 管）
  scenes/
    ch01.yaml         # 一个文件 = 一个场景，文件名 = 场景 id
sprite/
  xiaoman/*.png       # 立绘变体整图，文件名 ASCII，由角色注册表映射
voice/
  ch01/ch01_0010.ogg  # 语音文件，按约定命名，编译时自动绑定
voice.lock            # 编译器维护的语音清单 lockfile（提交进版本库）
```

## 4. 场景文件格式

### 4.1 场景头

```yaml
scene: ch01           # 场景 id，必须与文件名一致（编译器校验）
title: 放学后          # 可选，给编辑器/流程图用
steps: [ ... ]
```

### 4.2 steps 归一化规则（核心）

`steps` 是一个列表，每一项按以下顺序判别：

1. **纯字符串** → `narrate`（旁白/描述），**无语音**。
2. **映射，且含保留指令键**（见 4.3 指令集）→ 指令。一个 step 只允许一个主指令键 + 该指令的辅助键。
3. **映射，且首键是已注册角色名**（可带 `@表情` 后缀）→ `say`（台词），**默认有语音**。辅助键：`id` / `voice` / `face` / `at`。
4. 其他 → 编译错误。

冲突规则：角色名不得与保留指令键重名，在 `characters.yaml` 注册时报错。

台词的三种写法（语义等价，由编译器归一化）：

```yaml
- 小满: "你还不回家吗？"          # 最简形式
  id: ch01_0010                  # 语音 id 由工具分配后写回，与角色键平级

- 小满@赌气: "哼。"               # @表情 简写 → 归一化为 face 字段
  id: ch01_0020

- 小满: "（小声嘟囔了一句什么）"   # 显式关闭语音，无需 id
  voice: false
```

`face` 同时驱动两件事：切换该角色当前表情（参与立绘变体解析，见第 5 节），以及写入录音台本作为情绪标注。

### 4.3 指令集 v0.1

| 指令 | 形式 | 说明 |
|---|---|---|
| `narrate` | 纯字符串 | 旁白，无语音 |
| `say` | `角色名[@表情]: 文本` | 台词，默认有语音 |
| `text` | `text: 文本` 或 `text: { style, title, content }` | 全屏文字层（信件/读白/简介），见 14 节 |
| `shake` | `shake: 弱/中/强` 或 `shake: { intensity, ms }` | 全屏震动演出，见 17 节 |
| `item` | `item: { get/lose: 物品, n }` | 增减物品，见 15 节 |
| `money` | `money: { earn/spend: 数额 }` 或 `money: 数额` | 增减货币，见 16 节 |
| `bg` | `bg: 名称` 或 `bg: { name, transition, duration }` | 切背景 |
| `show` | `show: 角色名` 或 `show: { who, face, outfit, state, at, transition }` | 角色登场；**已在场则为更新**（只改给出的维度） |
| `hide` | `hide: 角色名` 或 `hide: { who, transition }` | 角色退场（退场不清空粘性维度） |
| `bgm` | `bgm: 曲名` / `bgm: stop` / `bgm: { name, fade }` | 背景音乐，`fade` 毫秒 |
| `se` | `se: 音效名` | 音效（不打断 BGM/语音） |
| `wait` | `wait: 毫秒` | 演出停顿 |
| `set` | `set: { 变量: 值或表达式 }` | 修改变量，见第 6 节 |
| `if` | `if: 表达式` + `then: [...]` + 可选 `else: [...]` | 条件分支（可嵌套） |
| `random` | `random: [加权分支...]` | 随机分支，见第 8 节 |
| `choice` | `choice: [选项...]` | 玩家选择，见第 7 节 |
| `label` | `label: 名称` | 场景内跳转锚点（场景内唯一） |
| `jump` | `jump: 目标` | 跳转：先解析为本场景 label，再解析为场景 id；歧义时编译报错 |
| `end` | `end: 结局id` | 到达结局（须在 story.yaml 注册），运行时记入跨周目存档 |

所有引用（角色、背景、立绘变体、BGM、SE、label、场景、结局）都在编译期解析并校验。

## 5. 角色立绘模型：维度寻址的整图变体

### 5.1 设计取舍

不做运行时图层合成（base+衣着+表情贴片）——AI 生成的图层对齐不可控。改为：**每张立绘是一张完整成品图，但在注册表中用语义维度标注**，脚本按维度描述角色状态，编译器解析到具体文件。组合受剧本约束，是有限集合而非乘法爆炸；剧本没用到的组合根本不需要出图。

### 5.2 寻址维度

| 维度 | 选择方式 | 粘性 | 脚本中如何改变 |
|---|---|---|---|
| `outfit` 衣着 | 单选 | 粘性（跨 step/场景保留，进存档） | `show: { who, outfit: 便服 }` |
| `state` 状态（淋湿/脏污/受伤…） | 单值或多值标签集 | 粘性 | `show: { who, state: 淋湿 }`，`state: []` 清空 |
| `face` 表情 | 单选 | 粘性，但最高频变化 | `@表情` 简写或 `show`/say 的 `face` |

变体解析 = 用角色当前 `(outfit, state集合, face)` 在注册表中查**精确匹配**。

### 5.3 注册表（characters.yaml）

```yaml
characters:
  小满:
    color: "#e8748a"
    voiced: true
    sprite:
      dims:                                     # 维度词表：先声明、后出图（脚本先行）
        outfit: [校服, 便服]
        state: [淋湿, 受伤]
        face: [默认, 微笑, 赌气, 失落]
      default: { outfit: 校服, face: 默认 }     # 首次登场的初始维度，state 默认为空
      variants:                                 # 可以为空数组——零出图也能跑（占位渲染）
        - { outfit: 校服, face: 默认, file: xiaoman/seifuku_normal.png }
        - { outfit: 校服, face: 微笑, file: xiaoman/seifuku_smile.png }
        - { outfit: 校服, face: 微笑, state: 淋湿, file: xiaoman/seifuku_smile_wet.png }
  我:
    voiced: false        # 无立绘角色：省略 sprite 即可
```

脚本里写中文维度值，文件名保持 ASCII（URL/跨平台安全），映射只存在于注册表。
`dims` 可省略——省略时词表由 `variants` + `default` 推导；显式声明的价值是在一张图都没出之前就提供拼写校验和编辑器自动补全。

### 5.4 词表与文件解耦：错误 vs 警告

| 情况 | 严重级 | 处理 |
|---|---|---|
| 脚本里的维度值不在词表中（如 `@徽笑`） | **错误** | 拼写错误，必须改脚本 |
| 可达组合没有注册变体 / 变体文件不在磁盘 | 警告 | 运行时占位渲染（见 12 节），照常调试 |
| 注册了但任何剧情路径都到不了的变体 | 提示 | 可能是死资产或剧本笔误 |

编译器同时导出**立绘生成清单**：所有可达但缺图的组合 + 触发它的场景/行号 + 剧情上下文——直接交给 AI 出图流程，与录音台本导出对称。出图完成放入 `sprite/`、补一行 variant 注册，警告即消除。

### 5.5 编辑素材与生产元数据（production）

资产分两类：**运行资产**（bg/bgm/se/sprite/voice，随游戏发布）与**编辑素材**（`production/` 目录，只服务于生产管线，不进 IR、不打包）。编辑素材挂在 `characters.yaml` 的角色节点下，与角色的数据关联天然成立：

```yaml
characters:
  小满:
    sprite: { ... }                # 运行资产
    production:                    # 编辑素材
      refs:                        # AI 出图参考立绘（角色设定图/三视图）
        - production/refs/xiaoman/character_sheet.png
      tts:                         # TTS 语音生成配置
        provider: gpt-sovits       # 服务标识，透传给生成接口
        sample: production/tts/xiaoman/sample.wav   # 音色参考音频
        params: { speaker: xiaoman_v1, speed: 1.0 } # 自由参数，原样透传
```

数据关联总图（注册表 = 关联中枢，目录约定 = 发现机制）：

```
脚本中的"名字" ──┬─ assets.yaml     backgrounds/bgm/se: 名字 → bg|bgm|se/文件
                ├─ characters.yaml  角色名 → sprite.variants:(outfit,state,face) → sprite/文件
                │                          → production.refs / production.tts（编辑素材）
                └─ 语音 id ──约定派生──→ voice/<场景>/<id>.ogg + voice.lock
生产管线：立绘清单 + refs → AI 出图 → sprite/ 落盘 → 注册 variant（警告消除）
          录音台本 + tts 配置 → TTS 接口 → voice/ 落盘 → voice.lock
```

- 编译器解析 `production` 仅做文件存在性提示（**info 级**，比资产警告更轻），其内容绝不写入 IR。
- 编辑器的资产管理面板基于此模型：磁盘扫描发现未注册文件 → 一键注册写回 YAML（结构化编辑、保留注释、走撤销栈）；参考图/音色文件 → 关联到角色的 production 块。

## 6. 状态与表达式

### 6.1 变量

- 所有变量必须在 `story.yaml` 的 `vars:` 中声明并给默认值，使用未声明变量是编译错误。
- 两个命名空间：
  - **单周目变量**（默认）：随存档保存，新开局重置为默认值。
  - **跨周目变量**：`global.` 前缀（如 `global.cleared_good`），持久化于本地，跨周目继承。结局达成由 `end` 指令自动写入。

### 6.2 表达式语言（刻意保持极小）

用于 `if`、选项的 `if`、`set` 的值、`random` 的权重。**绝不使用 eval**，编译期解析为 AST 存入 IR。

- 字面量：数字、`true`/`false`、单引号字符串 `'小满'`
- 变量引用：标识符（允许 CJK 字符）、`global.` 前缀
- 运算符：`+ - * /`、`== != > >= < <=`、`&& || !`、括号
- 随机函数（消耗 VM 内的种子 PRNG，见 8.2）：
  - `rand()` → `[0, 1)` 浮点
  - `randint(a, b)` → `[a, b]` 整数（含两端）

### 6.3 set 的取值规则（注意 YAML 陷阱）

```yaml
- set:
    met_xiaoman: true        # YAML 原生布尔/数字 → 字面量赋值
    favor: "favor + 1"       # 字符串 → 解析为表达式求值
    favor2: "favor += 1"     # += 语法糖，等价于上一行
    luck: "randint(1, 3)"    # 随机赋值
    name: "'小满'"            # 表达式里的字符串字面量要用单引号包一层
```

已知陷阱（实现采用 YAML 1.2）：`yes`/`no`/`on`/`off` 在 YAML 1.2 中是普通字符串而非布尔（Norway 问题不存在），但它们会流进表达式解析——编译器对此专门报错并提示改写 `true`/`false`（含 `voice: no` 这类位置）；`+1` 会被解析为数字 `1`，所以增量必须写成表达式字符串，不存在 `+1` 简写。

## 7. 选择支（choice）

```yaml
- choice:
    - text: "一起走吧，我送你。"
      if: "has_umbrella"            # 不满足则该选项不显示
      set: { favor: "favor + 1" }   # 选中后的即时效果
      goto: ch02_walk               # 跳转目标（label 或场景 id）
    - text: 再坐一会儿
      goto: sit_a_while
```

- 选项无 `goto` 时，选中后继续执行 choice 之后的下一个 step。
- 选项文本 v0.1 不配音（业界常态）。
- 编译器警告：若存在某种变量取值组合使所有选项都被 `if` 隐藏 → 潜在软锁。

## 8. 随机分支与确定性

### 8.1 random 指令

权重是表达式，运行时求值——**概率由变量+公式决定**：

```yaml
- random:
    - weight: 60
      then:
        - "雨幕里，路灯一盏盏亮了起来。"
    - weight: "favor * 40"          # favor 为 0 时该分支权重 0，不可能被抽中
      then:
        - "雨幕里，我又想起她跑开时的背影。"
    - weight: "10 + luck * 5"
      goto: rare_event              # 分支体与 choice 选项同构：then 或 goto
```

语义：求值各权重 → 负数按 0 计 → 按权重比例抽取。全部为 0 时取第一个分支并在开发构建发出运行时警告；编译器对"权重恒为 0 的分支"与"静态可知全部为 0"分别给出警告/错误。

### 8.2 确定性（关键约束）

- 随机数来自 **VM 状态内的种子 PRNG**（如 xoshiro）：新开局取一次熵作种子，此后每次 `rand`/`randint`/`random` 推进 PRNG 状态。
- **PRNG 状态随存档序列化**：读档后继续玩，掷点序列与存档时刻完全一致。
- **已掷出的结果进历史**：回滚（往回翻页）、backlog 回看展示的是已发生的结果，绝不重掷。
- 无头测试可注入固定种子，回放完全可复现；可达性分析将 random 的**所有分支视为可达**（权重表达式不参与静态剪枝）。

## 9. 语音模型

### 9.1 有无语音的判定

| 内容 | 默认 | 覆盖方式 |
|---|---|---|
| `narrate` 旁白 | 无语音 | 如需配音旁白，注册一个 `旁白` 角色改写为 say |
| `say` 台词 | 有语音 | 行级 `voice: false`；或角色级 `voiced: false`（如无语音主角） |
| 选项文本 | 无语音 | v0.2 再议 |

### 9.2 语音 ID 分配

- 格式：`<场景id>_<四位序号>`，如 `ch01_0010`。
- **按步长 10 分配**（0010、0020……），中途插行时取间隙号（如 0015），避免大面积重编号。
- **ID 一经分配永不复用**：删行后号码作废，不回收。
- ID 由编辑器/CLI 工具在保存时自动分配并**写回 YAML 源文件**（与角色键平级的 `id` 字段），写手不手填，但 ID 可见、可 diff。
- **不用内容哈希做 ID**——台词改稿后哈希变化会导致已录音频失联。文本哈希另有用途（见 9.3）。
- 音频文件按约定路径绑定：`voice/<场景id>/<语音id>.ogg`，无需在脚本中写文件名。

### 9.3 voice.lock 与三向核对

编译器维护 `voice.lock`（提交进版本库），记录每条语音的录制状态：

```yaml
ch01_0010:
  text_hash: "9f8a2c…"       # 录音时刻的台词文本哈希
  file: voice/ch01/ch01_0010.ogg
  duration_ms: 2840
```

每次编译做三向核对（**一律不报错**，缺语音由占位兜底，见 12 节）：

| 情况 | 严重级 | 处理 |
|---|---|---|
| 有台词、无音频文件 | 警告 | 占位：静音 + 按字数估算时长，自动模式照常推进 |
| 有音频、但台词文本哈希不匹配 | 警告 | 提示"已改稿，需重录"，照常播放旧音频 |
| 有音频、无对应台词（孤儿文件） | 提示 | 不打包 |

发布前的最终 QA 可用 `--strict` 开关把以上警告升级为报错，作为出厂检查，而非日常约束。

### 9.4 录音台本导出

编译器从 IR 一键导出录音台本（CSV/表格）：按角色分组，每行 = 语音 ID + 台词 + 情绪标注（face）+ 上下文（前后各两句）。录音回来的文件按 ID 命名放入 `voice/`，重新编译即完成绑定。

### 9.5 时长预计算

构建期读取音频文件时长写入 IR，运行时的自动模式计时、预加载调度、backlog 重听均直接使用。

## 10. 编译期校验清单

**结构层（JSON Schema）**：字段类型、必填项、指令形态（oneOf 判别）。

**语义层（linter，带行列位置）**：

总原则：**错误 = 脚本本身不自洽**（拼写、悬空引用、语法），改脚本才能消除；**警告 = 资产未到位**，占位兜底、清单驱动补齐，永不阻塞编译与调试。`--strict` 仅用于发布前 QA。

- 引用检查（错误）：未注册的角色/背景/BGM/SE/维度值、悬空的 jump/goto 目标、未注册的结局 id、重复 label
- 变量检查（错误）：未声明变量、表达式语法错误、类型不匹配
- 资产文件（警告）：已注册但文件缺失的背景/BGM/SE、可达组合缺立绘变体（占位渲染）、不可达的死变体（提示）、给不在场角色切 face（提示）
- 可达性：不可达的 step/场景/结局；choice 全选项可被隐藏的软锁风险；random 分支全部视为可达
- 随机：权重恒为 0 的分支（警告）、静态可知全部权重为 0（报错）
- 语音：ID 重复（错误）、三向核对（警告，见 9.3）
- YAML 陷阱（错误，属拼写类）：表达式/voice 字段里出现裸 `no`/`yes`/`on`/`off`，提示改写 `true`/`false`

## 11. IR 草案

编译产物为单个 JSON（或按场景分片）。示意：

```json
{
  "version": "0.1",
  "entry": "ch01",
  "scenes": {
    "ch01": {
      "ops": [
        { "op": "bg", "name": "home_morning" },
        { "op": "say", "who": "小满", "text": "你还不回家吗？", "face": null,
          "voice": { "id": "ch01_0010", "file": "voice/ch01/ch01_0010.ogg", "durationMs": 2840 } },
        { "op": "show", "who": "小满", "patch": { "state": ["淋湿"] } },
        { "op": "randomJump", "branches": [
            { "weight": ["lit", 60], "to": 12 },
            { "weight": ["*", ["var", "favor"], 40], "to": 15 } ] }
      ],
      "labels": { "sit_a_while": 9 }
    }
  },
  "sprites": { "小满": { "校服|淋湿|微笑": "sprite/xiaoman/seifuku_smile_wet.png" } },
  "lineIndex": { "ch01_0010": ["ch01", 1] }
}
```

要点：`if`/`choice`/`random` 的控制流在 IR 中扁平化为（条件/随机）跳转 + 索引；表达式为预解析 AST；立绘变体解析为"维度组合 → 文件"查找表；`lineIndex` 支持按语音 ID 反查（存档锚点、已读记录、backlog 重听）。

## 12. 运行时语义要点

- **VM 模型**：状态 = `{场景, 指令指针, 变量表, PRNG 状态, 舞台状态}`，舞台状态含背景、BGM 与每个角色的 `{在场与否, at, outfit, state, face}`，跨场景跳转保留。
- **存档**：序列化 VM 状态 + 最近一条语音行 ID 作为锚点；读档时若脚本已更新、指针失效，回退到锚点（或所在场景开头）。
- **推进节奏**：

| | 有语音行 | 无语音行 |
|---|---|---|
| 手动 | 点击推进；"打断语音 / 语音播完"做成玩家设置 | 点击推进 |
| 自动 | 音频播完 + 短停顿 | 按字数估算停留 |
| 文字呈现 | 即时全显（或按音频时长铺开） | 打字机效果 |

- **音频通道**：voice / BGM / SE 三通道独立音量；语音播放时 BGM ducking（压低 20–30%）；每角色语音音量微调。
- **占位渲染（缺失资产兜底，调试不中断）**：
  - 背景：按名称哈希取色的色块 + 居中显示资源名
  - 立绘：角色主题色剪影 + `角色名 / outfit / state / face` 维度标签
  - 语音：静音，时长按字数估算，自动模式节奏不变
  - BGM/SE：静默
  - 开发模式 HUD 常驻"缺失资产"面板，列出当前画面用到的占位项，点击可跳到对应清单条目
- **已读跳过**：以语音行 ID（无语音行用编译期生成的内部行 ID）为粒度记录已读。
- **预加载**：沿 IR 线性预取后续 N 句语音与演出资源；choice/random 处预取各分支首句。
- **无头模式**：VM 不依赖 DOM，可在 Node 中跑通全部分支（固定种子）——用于剧情测试（断言每个结局可达）与流程图生成。

## 14. 文本呈现 `text`（信件 / 读白 / 章节简介）

与对话框旁白 `narrate` 区分：`text` 是**全屏文字层**，点击/回车继续，默认无语音。

```yaml
- text: "雨下了一整夜。"                 # 最简形式 → note 风格
- text:
    style: letter        # letter 信纸 / monologue 读白 / intro 章节简介 / note 纯文本
    title: 致小满          # 可选抬头/标题
    content: |
      见字如面。
      等雨停了，我们一起回家。
```

- 控制流上是线性 op，可达性与立绘数据流不受影响。
- `style` 非法（不在四值内）报 `bad-instruction`。
- VM 产出独立的 `text` 事件（`{ style, title, content }`），播放器以 `.vn-textscreen` 覆盖层按风格渲染。

## 15. 物品系统 `item`

注册表 `story/items.yaml`（可缺省；缺省时无物品系统）：

```yaml
items:
  折叠伞:
    name: 折叠伞           # 显示名（缺省用 id）
    desc: 蓝色折叠伞。      # 物品栏描述
    image: umbrella.png   # item/umbrella.png，缺失占位渲染
    max: 1                # 可选堆叠上限
```

**核心模型**：物品是一个**只读数量命名空间 `item.<id>`**，求值为当前持有数量（默认 0，未注册 id 是编译错误，类型为数字）。增减用糖指令 `item:`，等价于对该命名空间做 `set`：

```yaml
- item: { get: 折叠伞 }                  # +1
- item: { get: 硬币, n: 5 }              # +5
- item: { lose: 硬币, n: 2 }             # -2（下限 0，归零即从库存移除）

- if: "item.折叠伞 >= 1"                  # 持有判断直接进分支
  then: [ "书包侧袋里，折叠伞还在。" ]

- choice:
    - text: 花 5 枚硬币买伞
      if: "item.硬币 >= 5"                # 数量参与选项可见性
      set: { item.硬币: "item.硬币 - 5", item.折叠伞: "item.折叠伞 + 1" }
```

- 库存 `Record<string, number>` 进 VM 状态与存档（旧存档缺字段降级为空库存）。
- 数量在 VM 内 `Math.max(0, floor(v))` 钳制，恒为非负整数。
- 播放器提供物品栏面板（菜单"物品"按钮），列出持有物的配图（缺失占位）、名称、数量、描述。
- 编辑器资产面板有"物品"分节：新增/编辑（名称·说明·堆叠上限）、配图（浏览本机拷入 `item/` 或换图）、
  注册 `item/` 下的散图、移除（可选同时删文件）、拖拽插入 `item: { get: <id> }`；写回 `items.yaml`
  走结构化编辑（保留注释、走撤销栈、保存才落盘），与背景/立绘注册流程一致。

## 16. 货币系统 `money`（可脚本开关）

`story.yaml` 的 `currency` 块控制开关：

```yaml
currency:
  enabled: true     # 缺省视为启用；显式 false 关闭
  name: 零钱
  symbol: ￥
  initial: 20       # 开局初始金额
```

- 启用时编译器注入**内置数字变量 `money`**（初值 `initial`），于是 `if: "money >= 100"`、
  `set: { money: "money - 20" }` 直接可用；关闭时引用 `money` 报"未声明"。
- 启用时若又在 `vars` 里声明 `money` → `money-var-conflict` 错误。
- 糖指令（金额在 VM 内钳制 ≥0；买不起由写手用 `if: money >= n` 自行守门）：

```yaml
- money: { earn: 50 }     # 赚钱
- money: { spend: 20 }    # 花钱
- money: 50               # 简写：正数 = 赚，负数 = 花
```

- 未启用 currency 却用 `money` 指令 → `money-disabled` 错误。
- 播放器在启用时常驻显示 `symbol + 金额`。

## 17. 全屏震动 `shake`

纯演出 Effect，非阻塞（与 `se` 同类），可放在任意位置（含 `bg`/`jump` 之后实现"切场景时震"）。

```yaml
- shake: 强                              # 弱/中/强（或 light/medium/heavy）
- shake: { intensity: heavy, ms: 600 }   # 完整形式，ms 缺省按强度取值
```

- 默认时长：弱 300ms / 中 500ms / 强 700ms。
- 不消耗 PRNG（确定性不受影响，无头测试照常记录 effect）。
- 播放器对 `.vn-stage` 套 CSS 抖动动画，强度控幅度；尊重 `prefers-reduced-motion`。
